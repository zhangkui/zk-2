# 效期计算说明

## 1. 核心公式

### 实际失效时间计算

```
实际失效时间 = min(原始有效期, 开封后有效期)
```

其中：
- **原始有效期**：物料包装上标注的失效日期，入库时录入
- **开封后有效期**：开封时间 + 物料配置的 `open_validity_days`（开封有效天数）
- 如果物料未开封，实际失效时间 = 原始有效期

## 2. 计算示例

### 场景一：未开封

| 字段 | 值 |
|------|-----|
| 原始有效期 | 2025-12-31 |
| 开封状态 | 未开封 |
| 开封有效天数 | 30天 |
| **实际失效时间** | **2025-12-31** |

### 场景二：开封后有效期短于原始有效期

| 字段 | 值 |
|------|-----|
| 原始有效期 | 2025-12-31 |
| 开封时间 | 2025-06-01 |
| 开封有效天数 | 30天 |
| 开封后有效期 | 2025-07-01 |
| **实际失效时间** | **2025-07-01** |

### 场景三：开封后有效期长于原始有效期

| 字段 | 值 |
|------|-----|
| 原始有效期 | 2025-06-15 |
| 开封时间 | 2025-06-01 |
| 开封有效天数 | 30天 |
| 开封后有效期 | 2025-07-01 |
| **实际失效时间** | **2025-06-15** |

## 3. 近效期判断

- **近效期**：当前时间 < 实际失效时间 ≤ 当前时间 + 预警天数（默认30天）
- **已过期**：实际失效时间 ≤ 当前时间

预警天数可通过系统配置 `warning_days_near_expiry` 修改。

## 4. 业务约束

1. 同一库存实例只能开封一次，后续更新状态时不会重新计算开封起点
2. 已过期物料禁止领用
3. 开封操作会触发状态重新计算和预警检查
4. 系统每小时自动扫描所有库存，更新状态并创建预警
