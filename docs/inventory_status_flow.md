# 库存状态流转说明

## 1. 库存状态定义

| 状态 | 标识 | 说明 |
|------|------|------|
| 正常 | normal | 库存充足，未开封，距效期较远 |
| 已开封 | opened | 已开封使用中 |
| 低库存 | low_stock | 数量 ≤ 物料最低库存阈值 |
| 近效期 | near_expiry | 实际失效时间在预警天数内（默认30天） |
| 已过期 | expired | 实际失效时间已过 |
| 已报废 | scrapped | 已执行报废操作，不可再使用 |
| 已用完 | used_up | 数量为0 |

## 3. 状态判定规则

状态判定优先级（从高到低）：

1. **已报废 (scrapped)**：永远保持，不可变更
2. **已用完 (used_up)**：数量 ≤ 0 且未报废
3. **已过期 (expired)**：实际失效时间 ≤ 当前时间
4. **近效期 (near_expiry)**：实际失效时间在预警天数内
5. **低库存 (low_stock)**：数量 ≤ 物料最低库存阈值
6. **已开封 (opened)**：已开封但不满足以上条件
7. **正常 (normal)**：默认状态

## 4. 触发状态变更的操作

| 操作 | 影响字段 | 触发状态重算 |
|------|----------|-------------|
| 入库 | quantity +N | 是 |
| 开封 | opened, open_time | 是 |
| 领用 | quantity -N | 是 |
| 归还 | quantity +N | 是 |
| 报废 | quantity -N, status | 是（报废后锁定） |
| 盘点 | quantity = 实际值 | 是 |
| 定时扫描 | - | 是（每小时） |

## 5. 业务约束

### 5.1 操作限制

| 操作 | 已过期 | 已报废 | 已用完 |
|------|--------|--------|--------|
| 开封 | ✗ | ✗ | ✗ |
| 领用 | ✗ | ✗ | ✗ |
| 归还 | ✓ | ✗ | ✓ |
| 报废 | ✓ | ✗ | ✓ |
| 盘点 | ✓ | ✗ | ✗ |

### 5.2 开封约束

- 同一库存实例只能开封一次
- 重复开封会返回错误："该库存已开封，不可重复开封"
- 开封时间一旦记录不可修改

### 5.3 报废约束

- 报废后状态锁定为 scrapped
- 报废后的库存不可再参与任何操作
- 支持部分报废（报废指定数量）
